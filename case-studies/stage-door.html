Stage Door Dance

Enhancing User Satisfaction, Usability, and Completion Rate for Dance Studio Website

Role UX/UI Designer, Researcher
Timeline 4 months
Tools Figma, Maze, Hotjar, Google Analytics

Project Overview

Stage Door Dance is a premier dance studio offering classes for all ages and skill levels. The studio needed a website redesign to improve user experience, increase class registrations, and better showcase their vibrant community and talented instructors.

The Challenge

The existing website faced several critical issues:

  • High bounce rate (68%) and low completion rate for class registration
  • Confusing navigation structure making it difficult to find class schedules
  • Poor mobile experience despite 65% of traffic coming from mobile devices
  • Outdated visual design that didn't reflect the studio's energetic brand
  • No clear call-to-action for trial class sign-ups

Research & Discovery

To understand user needs and pain points, I conducted comprehensive research:

User Surveys

45 current and potential students shared their experiences and expectations for the website

Usability Testing

8 participants attempted key tasks on the existing site, revealing critical usability issues

Analytics Analysis

Deep dive into Google Analytics and Hotjar data to identify user behavior patterns and drop-off points

Stakeholder Interviews

Conversations with studio owners and instructors to align business goals with user needs

Key Insight

Parents looking to enroll their children were frustrated by the inability to easily compare class times, instructor backgrounds, and pricing information in one place.

Key Learnings

  • Mobile-first is essential: Prioritizing mobile experience led to a cleaner, more focused design that worked well across all devices
  • Reduce cognitive load: Simplifying navigation and breaking complex tasks into steps significantly improved completion rates
  • Visual hierarchy matters: Clear visual hierarchy helped users quickly find the information they needed
  • Build trust through transparency: Providing detailed information about instructors and classes increased user confidence
  • Test early and often: Multiple rounds of testing with real users helped refine the design and catch issues before launch
